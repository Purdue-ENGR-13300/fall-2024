8.3.2. Task 2#

Text Encryption with Ciphers#

Learning Objectives:

  • Implement and apply classic encryption techniques such as the Atbash cipher.

  • Utilize file I/O functions in Python for reading and writing text data.

  • Develop an understanding of the concept of data encryption and its applications.

Background: Encryption remains a cornerstone of secure communication, both historically and in the modern digital world. By learning to implement classical encryption techniques such as the Atbash cipher, students can grasp foundational concepts of data security. Python, with its simplicity and robustness, serves as an excellent tool for exploring these encryption methods.

What is a Cipher? A cipher is a method for performing encryption or decryption—a series of well-defined steps that can be followed as a procedure. An algorithm, called a cipher algorithm, is used to transform the text to make it unrecognizable (encrypt it). The same or a different algorithm may be used to return the text to its original form (decrypt it).

How the Atbash Cipher Works: The Atbash cipher is a particular type of monoalphabetic cipher originally used to encrypt the Hebrew alphabet. It can be adapted for all alphabets and numeric systems. The Atbash cipher is a very specific case of a substitution cipher where the letters of the alphabet are reversed. In other words, the first letter becomes the last letter, the second letter becomes the second to last letter, and so on.

For the English alphabet, this would mean:

  • A is substituted with Z

  • B is substituted with Y

  • C is substituted with X … and so forth up to Z being substituted with A.

For digits, it similarly reverses the order:

  • 0 is substituted with 9

  • 1 with 8

  • 2 with 7 … continuing in this pattern.

Task: Develop a Python program named text_encryption.py that takes a text file named ‘secret.txt’ as input and applies the Atbash cipher to encrypt the message. The encrypted message should be written to an output file named ‘atbash_cipher.txt’.

Steps:

  1. Read the input text file ‘secret.txt’.

    • Utilize a function to read the content of the file, preparing it for encryption.

  2. Apply the Atbash cipher encryption:

    • Encrypt the content using the Atbash cipher, which involves reversing the alphabet for encryption.

    • Handle both alphabetic and numeric characters distinctly.

  3. Write the encrypted message to ‘atbash_cipher.txt’.

    • Use a function to write the encrypted message to a file, ensuring data is saved.

  4. Ensure proper error handling and user prompts for a smooth user experience.

    • Include error checks for file operations and ensure the user is informed of process completions and errors.

Functions#

The program will include the following key functions to handle various aspects of the encryption process. Each function is designed to be modular and reusable, enhancing ease of maintenance and potential for future extensions:

  1. read_file(filename):

    • Purpose: Reads a text file and returns its contents, ensuring that data is ready for encryption.

    • Inputs: A string representing the filename.

    • Outputs: A string containing the text from the file, which will be passed to the encryption function.

  2. atbash_cipher(text):

    • Purpose: Encrypts text using the Atbash cipher technique, which involves reversing the alphabet/numerical values to transform the plaintext into ciphertext.

    • Inputs: The plaintext string.

    • Outputs: A string representing the encrypted text. This includes converting all alphabetic characters according to their reversed positions in the alphabet and reversing numeric characters as well.

  3. write_file(filename, text):

    • Purpose: Writes the encrypted text to a file, ensuring that the output is saved for user review or further processing.

    • Inputs: The filename string where the encrypted text will be saved and the encrypted text string.

    • Outputs: None. The function writes directly to the file on disk, creating a tangible output from the encryption process.

In Python, the line reversed_alphabets = alphabets[::-1] uses slicing to reverse the string alphabets. Let’s break down the concept step-by-step:

Python Slicing Syntax

Slicing in Python is a way to extract portions of a sequence (like a string, list, or tuple). The syntax is: sequence[start:stop:step]

Where:

  • start: The starting index of the slice (inclusive).

  • stop: The ending index of the slice (exclusive).

  • step: The step size for slicing.

How Slicing Works in alphabets[::-1]

In the expression alphabets[::-1]:

  1. alphabets: This is the original string, "ABCDEFGHIJKLMNOPQRSTUVWXYZ".

  2. start: Omitted, which means the slicing starts from the beginning.

  3. stop: Also omitted, which means the slicing goes until the end of the string.

  4. step: Set to -1, meaning the slicing is done in reverse order.

Breaking It Down

When step is -1:

  • The slicing starts at the last character ('Z') and proceeds backward to the first character ('A').

  • The result is a new string with characters reversed: "ZYXWVUTSRQPONMLKJIHGFEDCBA".

Example

Let’s illustrate this with a smaller example:

  • Suppose we have a string alphabets = "ABC".

  • If you apply alphabets[::-1], the result will be "CBA".

Why This Works

The [::-1] slicing method is a concise way to reverse sequences in Python without writing a loop. This technique works for any sequence type (strings, lists, tuples) because slicing is a standard operation available to these data structures.

Conclusion

Using [::-1] is an efficient way to reverse a sequence. For the Atbash cipher, this approach provides an easy way to create a reversed version of the alphabet, which is then used to map each character directly to its reversed counterpart.
